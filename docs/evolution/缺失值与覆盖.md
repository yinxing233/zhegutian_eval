# Missingness 与 Coverage

早期系统使用：

normalized = total / available \* 100

进行归一化。

但随着语义评测变成可选项，
一个问题出现了：

只评格律的词，
可能比格律+语义双评的词，
得到更高标准化分数（normalized score）。

例如：

- A：格律满分，语义未评
- B：格律高分，语义也高分

系统可能显示：

A = 100
B = 95

虽然：

A 的观测覆盖率更低。

问题的本质是：

系统默认：

缺失（missing） = 不可见（invisible）

但实际上：

missing 本身就是信息。

例如：

- rule_skip
- judge_fail
- semantic_timeout

它们的含义完全不同。

于是系统开始显式暴露：

- coverage
- missing_dimensions
- missing_reason

并要求：

任何分数都必须结合 coverage 一起理解。

系统开始逐渐放弃：

单一总分

作为核心表达。

因为：

复杂系统中的“未观测”，
并不等于：

- 不存在
- 不重要
- 可忽略

它反而可能意味着：

当前观测系统的边界。
