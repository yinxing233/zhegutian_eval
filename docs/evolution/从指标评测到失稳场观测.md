# 从指标评测到失稳场观测

最初的评测系统，是按文学能力拆分评测指标的。

例如：

- 字数
- 平仄
- 押韵
- 对仗
- 语义

系统默认认为：

> 每个评测指标对应一种独立能力。

因此最早的修复逻辑也是点对点的：

- 字数不对 → 改字数
- 平仄错 → 修平仄
- 情感空 → 强化情绪表达

但随着 badcase 增多，一个问题开始出现：

同一个底层缺陷，会同时污染多个评测指标。

例如：

- “身份场崩塌”
  可能同时表现为：
  - 意象漂移
  - 情感失真
  - 语言模板化
  - 风格断裂

反过来：

同一个评测指标异常，也可能来自完全不同的根因。

例如：

“意象混乱”可能来自：

- 上下文窗口丢失
- 风格模板复读
- 语义中心塌缩
- 长程约束失效

这意味着：

评测指标本身并不是独立变量。

它们只是更深层结构失稳后的表面症状。

系统因此开始从：

指标评分（metric scoring）

转向：

不稳定性观测（instability observation）

后续引入：

- M 层（可观测症状指标 - Metric 层）
- F 层（latent instability hypothesis）

M 层负责记录：

- tone_fail
- rhyme_fail
- empty_output

  字数、平仄、押韵这三个是长期不会进行迭代的观测硬指标。

  对仗、语义指标根据诗词的不同，有可能不同。

  并且无论是人类还是大模型的指标评分，都是“不一定靠谱”的，这也导致了一致性评价问题。这是未来需要着重探索的方向。

F 层负责记录：

- F_identity
- F_imagery
- template_parroting

  同样，F层也是很难保证一致性问题的。
  后面也需要根据遇到的现实问题情况进行迭代。

其中：

- M 是观测结果
- F 是解释假说

两者不再混合。

`infer_instability_pattern()` 的职责，也不再是“判定模型属于什么问题”。

而是：

> 从多个耦合症状中，推测可能存在的底层失稳场。

这意味着评测系统的目标发生了变化。

系统不再试图回答：

> “模型得了多少分？”

而开始试图回答：

> “模型是在什么压力下，以什么方式开始失稳的？”
