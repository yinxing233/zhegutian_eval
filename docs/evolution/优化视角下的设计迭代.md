# 从"修 prompt"到"承认 provider 是不同实验动物"

**旧观念**：所有 provider 都是同一个 API 的不同皮肤，统一 prompt 就能统一行为。

**触发事件**：把 prompt 从高压改成低压后，DeepSeek 全线 reasoning 泄漏、GLM 正文污染加剧、Gemini 反而变差。

**新观念**：

> Gemini 有 Gemini 的行为生态，GLM 有 GLM 的行为生态，DeepSeek 有 DeepSeek 的行为生态。系统不负责统一它们，只负责稳定观测它们。

**落地**：建立了 `ProviderAdapter` 层，每个 provider 独立的 prompt_suffix 和 postprocess，不再互相污染。

# 从"清洗字符串"到"逆向隐式对话协议"

**旧观念**：GLM 输出的"格律说明""创作解析"是噪声，应该被清洗掉。

**触发事件**：修掉一种模式，它换一种形式回来。`extract_ci_text` 开始无限膨胀。

**关键判断**：

> 清洗的不是字符串，是模型的行为倾向。而行为倾向不是有限规则集合。等于正在逆向一个隐式对话协议。

**新观念**：

> 模型被 RLHF 训练成"完成任务 = 给答案 + 证明答案正确"。这个行为本身就是观测数据，不是需要被修掉的 bug。

**落地**：冻结 `extract_ci_text` 为粗粒度 salvage，不再穷举模式。残留污染交给 evaluator 打标签。

---

# 从"让所有 provider 可用"到"开始积累 failure data"

**旧观念**：系统要能稳定生成可评测正文才算 MVP。

**触发事件**：Gemini 恢复后，跑了一轮评测，`modern_04_topic` 正确触发 `template_parroting` + `aesthetic_entropy`，`stress_05_rhyme` 正确触发 `structure_incomplete` + `rhyme_fail`。

**新观念**：

> 继续改工程的收益迅速下降。failure pattern accumulation 开始变得更有价值。

**落地**：停止修清洗层、停止改 prompt、停止调 evaluator。开始收集 badcase，让标签分布倒逼 taxonomy。
