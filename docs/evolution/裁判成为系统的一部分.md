# 裁判成为系统的一部分

最初系统默认：

> evaluator（评测）是稳定可信的。

因此：

语义评测失败时，
`check_semantic()` 会直接返回：

```json
{ "score": 0.0 }
```

系统隐含认为：

没有得到有效结果 ≈ 模型表现极差

但后续观察发现：

大量 0 分并不是模型崩坏。

而是：

JSON 解析失败
API 异常
裁判输出不稳定
response format 崩坏

问题在于：

这些错误会进一步污染失稳分析。

例如：

semantic = 0
→ template_parroting
→ aesthetic_entropy
→ instability tags 污染

系统第一次意识到：

evaluator 本身也可能失稳。

因此：

“未观测” 与 “低质量”

必须被严格区分。

后续系统引入：

{
"success": false,
"score": null,
"error_type": "json_parse_failed"
}

并要求：

pipeline 不消费失败分数
semantic=None 时停止 F 层推断

这意味着：

评测系统开始进入二阶观测。

即：

系统不仅观测模型。

还开始观测：

观测器是否稳定
数据是否可信
缺失是如何产生的

missing 不再被视为：0

而被视为：unknown

缺失本身，也开始携带结构。
